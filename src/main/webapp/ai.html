RAG with Jakarta EE

Building Cloud-Native RAG Applications

Generative AI Primer

  • AI systems that create new content
  • Based on Large Language Models (LLMs)
  • Trained on vast datasets
  • Can generate text, code, images and more

What is RAG?

  • Retrieval-Augmented Generation
  • Combines LLMs with enterprise data
  • Enhanced accuracy through context retrieval
  • Reduced hallucinations ... or fabrications

Why RAG?

  • Cost-effective vs model fine-tuning
  • Up-to-date information without retraining
  • Maintains data privacy and control
  • Traceable responses (source documents)
  • Faster implementation cycles
  • Lower computational requirements

Architecture Overview

  • Payara Cloud for deployment
  • Neon DB as vector store
  • Azure Blob Storage for documents
  • LangChain4j for RAG operations
  • Zero-ops deployment model

Application Flow

  • Startup: Scan Azure Storage container
  • Process: Embed documents via LangChain4j
  • Storage: Save embeddings to Neon DB
  • Cleanup: Move processed files
  • Continuous operation until shutdown
